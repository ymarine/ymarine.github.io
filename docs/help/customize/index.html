Docsメニュー備忘録

ページ左側のDocsメニューの動きが想定とだいぶ違っていたため備忘録としてメモ

公式(と他の方が書いた日本語のブログ)の説明から予想された動き #

\config_default\menus\menus.ja.toml

の[[docs]]部分によりDocs部分のメニュー内容や表示順が決まる

[[docs]]
  name = 表示名
  weight = 表示順(数字の小さい方が上)
  identifier = 下位の階層から指定する際に使う
  url = Docsの該当する部分を押したらこのURLに飛べる

例えば

[[docs]]
name = "プロローグ"
weight = 10
identifier = "prologue"
url = "/docs/prologue/"

[[docs]]
name = "レシピ"
weight = 20
identifier = "recipes"
url = "/docs/recipes/"

[[docs]]
name = "ヘルプ"
weight = 60
identifier = "help"
url = "/docs/help/"

の場合、

>プロローグ
 >下位階層
>レシピ
 >下位階層
>ヘルプ
 >下位階層

となると予想

しかし実際は

>PROLOGUE
 >下位階層
>HELP
 >下位階層
>RECIPES
 >下位階層

並び順、メニュー名が想定と違う

また、現ファイル

\docs\help\DocsMemo.md

階層指定

menu:
  docs:
    parent: "recipes"

各ドキュメントファイルの上記部分で入れたい階層のidentifierを指定することでフォルダ階層に関わらず入れたい階層の下位に入れることが出来ると考えられたが、想定通りの位置に入らない

実際の動き #

並び順 #

各フォルダ(\docs\prologueなど)にある各_index.mdファイルの中のweight: XXX部分が関係している

  • weightの数字が大きいほど上、小さいほど下に表示される
  • weightが未設定の場合は優先的に上に表示される
  • weight未設定の階層が複数ある場合は作成日時の古いものが上に表示される

メニュー名 #

  • 各_index.mdファイルのtitle: "●●●"部分が反映される
  • titleに英小文字を入力した場合は自動的に大文字に変換される

例

\docs\recipes\_index.md

の中で、title: "れしrecipe"と指定

表示名

>れしRECIPE
 >下位階層

階層 #

現ファイル

\docs\help\DocsMemo.md

階層指定

menu:
  docs:
    parent: "recipes"

想定

>レシピ
 >Docs部分備忘録

実際

>ヘルプ
 >Docs部分備忘録
  • 各ファイルの階層指定が反映されておらず、ファイルを入れたフォルダ通りの階層になっているようです。

その他 #

URL #

URLは

http://localhost:1313/docs/help/docsmemo/

フォルダ名、ファイル名がそのまま適用される模様

ただし、ファイル上部の設定部分に slug: カスタマイズしたいURL を追加するとファイル名部分を好みのURLに変更可能

例

slug: customize

URL

http://localhost:1313/docs/help/customize/

その場合も上のURLは同様に使用可能(リンクが切れることはない)

Docsメニューからのリンクで移動するページがカスタマイズしたURLになる

下部にあるページ遷移 #

ページ下部にある各ページを順々に遷移しているリンクについて #

  • 各ページのweight: XXXの部分が関係している
  • フォルダ階層に関係なく、全ページに対して順番を設定し、その通りに遷移している
  • weight: XXXの数字が小さい方が順番が早い(左側に表示)
  • weight: XXXの数字が大きい方が順番が遅い(右側に表示)
  • weightの値が未設定の場合は順番が遅くなる(最大数値のweight: 999より後に表示)
  • weightの数値が同じ場合は、作成日が新しいファイルの方が順番が早くなる

※Docsメニュー(_index.md)のweightの値とは異なる動きをするため注意

ページに画像を表示したい場合 #

  • mdファイルと同じ階層に画像ファイルを設置したい場合は、フォルダを設置し、その中にindex.mdという名前で本文ファイルを作らなければならない

詳しくはリンク先

※staticフォルダに画像を格納し、そこから取得する方法も試したが、うまくいかない。 そのため、現状では、この方法以外で画像を表示する方法が分からない。
